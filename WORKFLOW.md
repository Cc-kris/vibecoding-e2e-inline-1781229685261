# Workflow

This repository is managed by the vibecoding-dashboard automated development pipeline.

## Branch Naming

- Task branches: `task/<task-id>-<timestamp>`
- Feature branches: `feat/<short-description>`

## PR Process

1. Branches and PRs are auto-generated by the dashboard's AI agents.
2. Each PR maps to a TaskSpec (with explicit allowedChangeScope).
3. Code review is performed by the AI review agent first, then optionally by humans.
4. CI must pass before merge is allowed.
5. Merge requires explicit user confirmation through the dashboard dialogue.

## Restrictions

- Direct pushes to `main` are forbidden — all changes flow through PRs.
- Each change set is scope-checked against the TaskSpec.allowedChangeScope.
- Auto-merge is disabled; merges require dialogue confirmation.
